## What is Echoseo?

Echoseo is a self-hosted SEO discovery tool designed for freelancers, hobbyists and small creators who want to understand and improve their online visibility. You enter a username, email address, account name or any other online identifier, and Echoseo searches the web across multiple sources, checks for data breaches via Have I Been Pwned, and uses a local LLM to bundle everything into a clean, readable report — complete with a vindbaarheid (findability) score from 0 to 100 and personalised improvement suggestions.

The name **Echoseo** is a blend of *Echo* — the idea of the web reflecting your identity back at you — and *SEO*, the core concept of online findability and visibility.

---

## Core Features

- **Multi-source search** — queries multiple search engines (Google, DuckDuckGo, Brave, Bing) simultaneously and aggregates the results into a unified hit list
- **Findability score 0–100** — a hybrid score calculated from hard signals (number of hits, platform quality, domain diversity) combined with qualitative labels generated by a local LLM
- **LLM-generated report** — a local language model (hosted on-device, no cloud dependency) reads the aggregated search results and writes a short, human-readable summary in plain Dutch or English
- **Improvement suggestions** — if your score falls below a user-defined threshold, Echoseo automatically generates concrete, actionable tips to improve your online presence
- **Have I Been Pwned integration** — checks whether your email address appears in any known data breaches and factors the result into your score and report
- **Fully self-hosted** — everything runs on your own hardware (designed for Raspberry Pi 5 + Ollama, but works on any machine running Node.js and a local LLM endpoint)
- **Privacy-first** — no data is sent to third-party services beyond the search APIs and HIBP check; logs are minimal and configurable

## Scoring Model

The findability score is calculated deterministically in the backend based on the following components:

| Component | Max points | Description |
|-----------|-----------|-------------|
| Hit volume | 40 pts | Number of unique hits across all search engines |
| Platform quality | 30 pts | Presence on high-value platforms (own domain, LinkedIn, GitHub, etc.) |
| LLM quality label | 20 pts | Qualitative label assigned by local LLM based on result content |
| HIBP breach penalty | −20 pts | Deducted based on number of known data breaches |

The LLM is asked to return structured labels (`vindbaarheid_kwalitatief`, `kwaliteit_profielen`, `risico_niveau`) — never a raw number — which are then mapped to points by the backend. This keeps the score transparent, reproducible and tweakable without touching the model.

## Tech Stack

| Layer | Technology |
|-------|------------|
| Frontend | HTML / CSS / Vanilla JS (prototype), later React or HTMX |

## Getting Started

### Prerequisites

- Node.js >= 18.0.0
- [Ollama](https://ollama.com) installed and running locally
- API keys for the search engines you want to use (see `.env.example`)
- A free [Have I Been Pwned API key](https://haveibeenpwned.com/API/Key)

### Installation

```bash
# Clone the repository
git clone https://github.com/Nihilago/Echoseo.git
cd Echoseo

# Install dependencies
npm install

# Copy and fill in your environment variables
cp .env.example .env

# Pull the LLM model via Ollama
ollama pull phi3:mini

# Start the backend
npm start
```

Then open `http://localhost:3000` in your browser.

### Environment Variables

```env
# Search APIs
GOOGLE_API_KEY=your_google_api_key
GOOGLE_CX=your_custom_search_engine_id
BING_API_KEY=your_bing_api_key
BRAVE_API_KEY=your_brave_api_key

# Have I Been Pwned
HIBP_API_KEY=your_hibp_api_key

# Local LLM
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=phi3:mini

# Score threshold default (user can override via UI)
DEFAULT_SCORE_THRESHOLD=60

# Server
PORT=3000
```

## Privacy & Legal

Echoseo is designed with privacy in mind:

- No user queries are stored permanently by default
- The LLM runs fully locally — no data leaves your machine for AI inference
- Search API calls are made server-side; no identifiers are exposed in client-side requests
- HIBP uses k-anonymity for password checks
- Users are responsible for complying with GDPR/AVG when using Echoseo to look up information about other people
